## 📅 운영 방식

- **일시**: 매주 금요일 점심시간 (오프라인)
- **방식**: 각자 정해진 문제를 풀고 **Pull Request(PR)**를 생성하여 서로의 코드를 리뷰합니다.
- **분량**: 매주 정해진 알고리즘 테마의 **골드 4~5 수준** 문제 2개

<br>

## 📂 폴더 구조

레포지토리의 폴더 구조는 아래와 같습니다. PR 생성 시 경로를 꼭 확인해주세요.

\`month / week / name / [platform]_[problem_number].java\`
- 예시: \`02/week1/장지인/BOJ_1234.java\`

<br>

## 💬 커밋 & PR 컨벤션

### Commit
> \`[platform] 문제번호 문제명 - 이름\`
- 예: \`[BOJ] 1158 요세푸스 문제 - 장지인\`

### PR Title
> \`[월 주차] 알고리즘 테마 - 이름\`
- 예: \`[2월 1주차] Linked List & Tree - 장지인\`

### PR Body
PR 본문에는 오프라인 리뷰 시 소통을 원활하게 하기 위해 아래 내용을 포함합니다.

> ## 문제 요약
> 접근 방식 및 알고리즘 분류
>
> ## 어려웠던 점
> 풀면서 막혔던 부분이나 고민했던 로직
>
> ## 공유하고 싶은 팁
> 새롭게 알게 된 문법이나 최적화 방법
